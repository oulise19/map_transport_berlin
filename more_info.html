More on traffic data used for the website

Pipeline used for Telraam data

Data sources

In this visualization, we present data from two different sources. The Geoportal data comes from the Geoportal of Berlin. No processing was applied to it, except for car traffic, which was calculated by subtracting dtvw_lkw (heavy vehicles only) from dtvw_kfz (all vehicles).

The second dataset comes from the Telraam platform, which uses sensors to collect raw counts of cyclists, pedestrians, cars, and heavy vehicles. This data is accessible via an API or the Berlin Zählt Mobilität website, though only as raw hourly counts.

Processing methodology

DTVw — Durchschnittlicher Tagesverkehr werktags (Average Daily Traffic on workdays): the average number of vehicles and cyclists passing a road section on a typical weekday, excluding public holidays.

To bring Telraam data closer to this metric, we processed the raw counts by calculating the average number of vehicles and cyclists per segment on a typical weekday. This was done in Python using Pandas — grouping by segment, weekday, and hour, then averaging.

Public holidays were excluded using the holidays library. School holidays, which have no equivalent library, were excluded manually using dates from the official Berlin website.

Limitations

The processed Telraam data is more comparable to Geoportal data, but it's still not as comprehensive — it only covers segments with installed sensors, and may vary due to sensor placement and calibration. It should be interpreted with caution, alongside other data sources.

Survey

The survey was written by Marek Stierts, with responses collected by students from Humboldt University Berlin. Two locations were chosen: one near the A100 extension, and another in Adlershof, close to the university campus.

The survey was conducted as a questionnaire. Some participant answers and identifying information are not included in this visualization.
